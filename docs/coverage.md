# Test Coverage Dashboard

ForgeKit maintains automated test coverage tracked on every commit. The report below is generated by the [Coverage Report CI workflow](https://github.com/SubhanshuMG/ForgeKit/actions/workflows/coverage.yml) and updated automatically on every push to `main`.

## About the Coverage Setup

ForgeKit uses **Jest** with `ts-jest` to test the CLI package. Coverage is collected with Istanbul and reported in four dimensions: lines, statements, branches, and functions.

### Thresholds

The CI will fail if coverage drops below these minimums:

| Metric | Minimum |
|--------|---------|
| Lines | 60% |
| Functions | 60% |
| Branches | 50% |

### Running coverage locally

```bash
cd packages/cli
npm test -- --coverage
```

This generates an interactive HTML report at `packages/cli/coverage/lcov-report/index.html`.

### Adding tests

Tests live in `packages/cli/src/__tests__/`. Each new feature or bug fix should include a corresponding test file.

See [CONTRIBUTING.md](/contributing) for the full testing guide.
