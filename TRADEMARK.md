# ForgeKit Trademark Policy

**"ForgeKit" and the ForgeKit logomark are trademarks of the ForgeKit project maintainers.**

The Apache License 2.0 under which ForgeKit source code is distributed does **not** include a license to use the ForgeKit trademark. This policy defines what is and is not permitted.

---

## What You CAN Do

- Fork the code and build your own tool, under the terms of Apache 2.0
- Say "based on ForgeKit" or "powered by ForgeKit" to accurately describe origin
- Write about ForgeKit in articles, blog posts, talks, and documentation
- Use the name in accurate comparisons (e.g. "similar to ForgeKit")
- Contribute back under your own name with proper attribution

---

## What You CANNOT Do

- Name your fork or derived project "ForgeKit", "ForgeKit Pro", "ForgeKit Cloud", or any confusingly similar name
- Use the ForgeKit logomark (the geometric F mark) or any visual variation without written permission
- Create a website, product, or service that implies official ForgeKit affiliation
- Register domain names containing "ForgeKit" for a competing or related product
- Use "ForgeKit" in a company name, product name, or npm package name without written permission
- Remove or alter copyright notices, the NOTICE file, or SPDX headers from source files
- Republish this software on npm or any package registry under a name implying it is the official ForgeKit package

---

## Forks and Derivatives

If you fork ForgeKit:

1. **Rename your fork** — it must not be confused with the official project
2. **Credit the origin** — your README must state it is a fork of ForgeKit and link to https://github.com/SubhanshuMG/ForgeKit
3. **Keep copyright intact** — you must preserve all copyright notices, NOTICE file, and SPDX headers (required by Apache 2.0)
4. **Do not impersonate** — do not represent your fork as the official ForgeKit project in any channel

Failure to comply with Apache 2.0 attribution requirements voids your license to use this software.

---

## Reporting Violations

If you believe someone is violating this trademark policy or misrepresenting a fork as the official ForgeKit:

### 1. Report to GitHub

- **Trademark infringement**: https://support.github.com/contact/trademark-policy
- **DMCA / copyright takedown**: https://github.com/contact/dmca
- **Impersonation**: https://support.github.com/contact/impersonation

When filing, reference:
- Original repository: https://github.com/SubhanshuMG/ForgeKit
- npm package: https://www.npmjs.com/package/forgekit-cli
- Official site: https://forgekit.build

### 2. Report to npm

If someone republishes this package under a confusingly similar name:
- npm support: https://www.npmjs.com/support
- npm security: security@npmjs.com

### 3. Open an Issue

Open an issue in this repository tagged `trademark-violation` with details of the infringement. Maintainers will review and escalate as needed.

---

## Requesting Permission

To request permission for uses not covered above, open a GitHub Discussion tagged `trademark-request` or contact the maintainers directly via GitHub.

We are generally friendly about legitimate uses — just ask first.

---

## Why This Policy Exists

Trademark protection protects the community: it ensures that when someone uses ForgeKit, they know they're getting the real thing, maintained to this project's standards of quality and security.

This policy is modeled after the trademark policies used by the Linux Foundation, Kubernetes, and Node.js projects.
